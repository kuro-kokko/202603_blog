> 対象: blog 静的サイトジェネレーター

## 1. FrontMatter パース / バリデーション

### 1.1 必須フィールド
- [x] `title` が存在する場合、正しく取得できる（1-1）
- [x] `tags` が正しく取得できる（1-2）
- [x] `updatedDateTime` が `Date` 型として正しく取得できる（1-3）

### 1.2 省略フィールドのデフォルト値
- [x] `updatedDateTime` を省略した場合、`createDateTime` と同じ値になる（1-8）
- [x] `tags` を省略した場合、空配列 `[]` になる（1-6）
- [x] `draft` を省略した場合、`false` として扱われる（1-7）
- [x] `description` を省略した場合、本文冒頭 140 文字が自動生成される（2-1）
- [x] `slug` を省略した場合、ファイル名から生成される（1-9）

### 1.3 slug 生成
- [x] `slug` をフロントマターで指定した場合、そのまま使われる（1-5、1-10）
- [x] `slug` を省略した場合、`relativePath` から `.md` を除いた値になる（例: `posts/hello-world`）（1-9）

### 1.4 description 自動生成
- [x] 見出し（`##` 等）は description から除去される（2-2）
- [x] Markdown 記法（`**`, `_`, `` ` `` 等）は除去される（2-2）
- （未実装） wiki リンク（`[[slug]]`）はリンクテキストに展開される
- [x] 141 文字以上の本文は 140 文字で切り捨てられる（2-3）

## 2. Markdown パース

### 2.1 基本変換
- [x] 見出し（`##`）が `<h2>` に変換される（11-1）
- [x] 段落が `<p>` に変換される（HTML出力に含まれる）
- [x] 箇条書き（`-`）が `<ul>` / `<li>` に変換される（11-2）
- [x] 番号付きリスト（`1.`）が `<ol>` / `<li>` に変換される（11-3）
- [x] コードブロック（` ``` `）が `<pre><code>` に変換される（11-4）
- [x] インラインコード（`` ` ``）が `<code>` に変換される（11-4）
- [x] 太字（`**`）が `<strong>` に変換される（11-5）
- [x] リンク（`[text](url)`）が `<a>` に変換される（11-6）
- [x] 画像（`![alt](url)`）が `<img>` に変換される（11-7）

### 2.2 wiki リンク変換
- [x] `[[slug]]` が `<a href="/slug/">slug</a>` に変換される（5-3）
- [x] `[[slug|エイリアス]]` が `<a href="/slug/">エイリアス</a>` に変換される（5-4）
- [x] 存在しない slug への wiki リンクもリンクとして変換される（12-1、12-2、12-3）

### 2.3 見出し ID（ToC アンカー用）
- [x] `## 見出し` に `id` 属性が付与される（6-2）
- （手動） 同一テキストの見出しが複数ある場合、一意な ID が生成される

### 2.4 XSS 対策
- [x] `<script>` タグを含む Markdown はサニタイズされて無害化される（9-1）
- [x] `<img onerror="...">` 等のイベントハンドラ属性は除去される（9-2）

## 3. BuildContext 構築

### 3.1 TagMap
- [x] 記事のタグが正しく TagMap に登録される（8-1）
- [x] 複数記事に同じタグがある場合、両方が TagMap の同一キーに入る（8-2、8-3）
- （手動） タグなし記事は TagMap に現れない

### 3.2 BacklinkMap
- [x] `[[slug]]` で参照された記事の `backlinks` に参照元が追加される（5-2）
- [x] 双方向リンク（A→B、B→A）の場合、両方に backlinks が設定される（15-1、15-2）

### 3.3 CalendarMap
- [x] `calendar.enabled: true` のとき CalendarMap が構築される（7-1）
- [x] 記事が `"YYYY-MM"` をキーとして正しく分類される（7-1）

### 3.4 draft 除外
- [x] `draft: true` の記事は `pages` に含まれない（3-1）
- [x] `draft: true` の記事は TagMap に含まれない（3-3）
- （手動） `draft: true` の記事は BacklinkMap / CalendarMap にも含まれない

### 3.5 アンダースコア除外
- [x] `_` で始まるファイルは `excludeUnderscored: true`（デフォルト）のとき除外される（3-4）
- （未実装） `excludeUnderscored: false` に変更するとアンダースコアファイルも処理される

## 6. 検索（クライアントサイド）

### 6.1 トークナイズ
- （手動） 日本語テキストがバイグラム + ユニグラムに分割される（例: `"ブログ"` → `["ブ", "ロ", "ブロ", "ロ", "グ", "ログ"]`）
- （手動） ASCII テキストは小文字化・単語単位で分割される
- （手動） インデックス構築時とクエリ時で同じトークナイザーが使われる

### 6.2 検索結果
- （手動） タイトルに含まれるキーワードでヒットする
- （手動） description に含まれるキーワードでヒットする（例: `"ブログ"` で `"このブログの…"` がヒット）
- （手動） タグ名でヒットする
- （手動） タイトルヒットが description ヒットより高スコアになる（`boost: { title: 3 }`）
- （手動） 入力が空のとき結果が消える
- （手動） ヒット 0 件のとき「該当する記事がありませんでした」と表示される

### 6.3 UI
- （手動） 入力から 200ms のデバウンス後に検索が実行される
- （手動） URL の `?q=xxx` パラメータがある場合、ページロード時に自動検索される
- （手動） 検索結果にタイトル・description・タグが表示される
