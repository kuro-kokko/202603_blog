```
│  ビルド環境（Node.js）                            │

│  content/*.md ──→ parseDirectory()               │
│                       ↓                           │

│                       ↓                           │

│                       ↓                           │
│                   out/ (静的ファイル)              │

│                                                   │
│  HTML + CSS ──→ ページ表示                        │
│  search-index.json + search.client.js → 全文検索 │
└─────────────────────────────────────────────────┘
```

## 3. ビルドフロー

```
npm run build
    ↓
src/build.tsx
    │
    ├─ 1. パース
    │       parseDirectory('content/', config)
    │           ├─ .md ファイルを再帰収集
    │           ├─ gray-matter でフロントマター分離
    │           ├─ resolveFrontMatter() でデフォルト値補完
    │           ├─ extractDescription() で説明文生成
    │           ├─ extractWikiLinks() でリンク先 slug 収集
    │           └─ unified パイプラインで hast 生成
    │                   remarkParse → remarkWikiLink
    │                   → remarkRehype → rehypeSlug
    │                   → rehypeSanitize
    │           → Page[]
    │
    ├─ 2. コンテキスト構築
    │       buildContext(pages, config)
    │           ├─ buildTagMap()
    │           ├─ buildBacklinkMap()
    │           ├─ buildCalendarMap()
    │           └─ page.backlinks への注入
    │           → BuildContext
    │
    ├─ 3. 出力先初期化
    │       out/ ディレクトリをクリーン → 再作成
    │
    ├─ 4. アセットコピー
    │       assets/ → out/assets/
    │
    ├─ 5. ページ HTML 生成
    │       IndexPage        → out/index.html
    │       ArticlePage × N  → out/{slug}/index.html
    │       TagPage × M      → out/tags/{tag}/index.html
    │       TagsPage         → out/tags/index.html
    │       SearchPage       → out/search/index.html
    │       DatePage × K     → out/date/{YYYY-MM-DD}/index.html
    │                          （同日複数記事の場合のみ）
    │
    └─ 6. 検索インデックス生成
            → out/search-index.json
```

## 4. モジュール設計

### 4.1 `src/lib/parser.ts`

Markdown ファイルを `Page` オブジェクトに変換する。

**主要関数:**

| 関数 | 入力 | 出力 | 役割 |
|------|------|------|------|
| `parseDirectory(dir, config)` | ディレクトリパス, SiteConfig | `Promise<Page[]>` | 再帰的にファイルを収集・パース |
| `parseFile(filePath, config)` | ファイルパス, SiteConfig | `Promise<Page \| null>` | 1 ファイルを Page に変換（除外対象は null） |
| `resolveFrontMatter(fm)` | 生 FrontMatter | `ResolvedFrontMatter` | 省略フィールドにデフォルト値を補完 |
| `extractDescription(rawContent)` | Markdown 本文 | `string` | 本文冒頭 140 文字を抽出（記法除去） |
| `extractWikiLinks(rawContent)` | Markdown 本文 | `string[]` | `[[slug]]` 形式のリンク先 slug 一覧 |

**unified パイプライン:**

```
remarkParse          Markdown テキスト → mdast
remarkWikiLink       [[slug]] → リンクノードに変換
remarkRehype         mdast → hast（allowDangerousHtml: false）
rehypeSlug           h2/h3 に id 属性を付与
rehypeSanitize       XSS サニタイズ（id/className を許可）
```

### 4.2 `src/lib/context.ts`

`Page[]` からビルド全体で共有するインデックスを構築する。

| 関数 | 出力 | 説明 |
|------|------|------|
| `buildContext(pages, config)` | `BuildContext` | TagMap・BacklinkMap・CalendarMap を構築し、各 Page の `backlinks` に注入する |

**インデックス構造:**

| 名称 | 型 | キー | 値 |
|------|----|------|-----|
| TagMap | `Map<string, Page[]>` | タグ名 | そのタグを持つ記事一覧 |
| BacklinkMap | `Map<string, Page[]>` | slug | そのスラッグへリンクしている記事一覧 |
| CalendarMap | `Map<string, Page[]>` | `"YYYY-MM"` | その月の記事一覧 |

### 4.3 `src/lib/remark-wiki-link.ts`

remark プラグインとして unified パイプラインに組み込まれる。

- テキストノード内の `[[slug]]` / `[[slug|ラベル]]` を検出
- `{ type: 'link', url: '/{slug}/', children: [text(label)] }` ノードに変換
- 存在しない slug へのリンクも変換する（リンク切れ検出は行わない）

### 4.4 `src/lib/toc.ts`

hast（HTML AST）から目次項目を抽出する。

```typescript
extractToc(hast: Root): TocItem[]
// h2・h3 要素を走査し、id 属性とテキストを収集
// depth: 2（h2）または 3（h3）
```

### 4.5 `src/lib/renderer.ts`

Preact コンポーネントを HTML 文字列に変換する。

```typescript
renderPage(vnode: VNode): string
// → '<!DOCTYPE html>\n' + preact-render-to-string の結果
```

### 4.6 `src/client/search.client.ts`

ブラウザで動作するクライアント検索スクリプト。esbuild でバンドルして `assets/search.client.js` として配置される。

**処理フロー:**
1. `DOMContentLoaded` で入力フォームと結果エリアを取得
2. `input` イベントを 200 ms デバウンスして検索実行
3. 初回検索時に `/search-index.json` を非同期取得して MiniSearch に読み込み（以降はキャッシュ）
4. URL パラメータ `?q=` があれば初期値としてフォームに設定し自動検索

**CJK トークナイザー:**

```
入力文字列
    ↓
CJK 文字の有無を判定
    ├─ CJK: ユニグラム + バイグラムを生成
    │       "ブログ" → ["ブ", "ブロ", "ロ", "ログ", "グ"]
    └─ ASCII: 小文字化して空白・記号区切りで分割
```

インデックス構築時とクエリ時で同じトークナイザーを使用し、一致精度を保つ。

## 5. URL 設計

| ページ種別 | URL パターン | 生成条件 |
|------------|-------------|---------|
| トップ | `/` | 常に生成 |
| 記事 | `/{slug}/` | 記事ごとに生成 |
| タグ別 | `/tags/{tag}/` | タグごとに生成 |
| タグ一覧 | `/tags/` | 常に生成 |
| 日付別 | `/date/{YYYY-MM-DD}/` | 同日複数記事の場合のみ |
| 検索 | `/search/` | 常に生成 |
| 検索インデックス | `/search-index.json` | 常に生成 |

slug はフロントマターで指定しない場合、ファイルの相対パスから `.md` を除いた値になる。
例: `content/posts/hello-world.md` → slug `posts/hello-world` → URL `/posts/hello-world/`

## 7. 設定項目（SiteConfig）

`config.ts` に定義する。

| フィールド | 型 | デフォルト | 説明 |
|------------|-----|----------|------|
| `title` | `string` | 必須 | サイト名。ページタイトルの末尾に付与される |
| `baseUrl` | `string` | 必須 | サイトの公開 URL（OGP タグの `og:url` に使用） |
| `description` | `string?` | — | サイト説明文 |
| `language` | `string?` | `"ja"` | `<html lang>` 属性の値 |
| `profile.enabled` | `boolean` | — | プロフィールカードの表示有無 |
| `profile.name` | `string` | — | 著者名 |
| `profile.bio` | `string?` | — | 自己紹介文 |
| `profile.avatar` | `string?` | — | アバター画像パス（省略時はイニシャル表示） |
| `profile.links` | `ProfileLink[]?` | — | 外部リンク一覧（GitHub・SNS 等） |
| `calendar.enabled` | `boolean` | — | カレンダーの表示有無 |
| `calendar.dateField` | `string?` | `"createDateTime"` | カレンダーの基準日フィールド |
| `build.outDir` | `string?` | `"out"` | 出力ディレクトリのパス |
| `build.excludeDrafts` | `boolean?` | `true` | `draft: true` の記事を除外するか |
| `build.excludeUnderscored` | `boolean?` | `true` | `_` 始まりのファイルを除外するか |
| `backlinks.position` | `"below" \| "sidebar"?` | `"below"` | バックリンクの表示位置 |

## 9. セキュリティ設計

### 9.1 サーバーサイド XSS 防止

Markdown パイプラインの末尾で `rehype-sanitize` を適用する。

- `allowDangerousHtml: false`（remark-rehype）により生 HTML はテキストノードに変換
- `rehypeSanitize` の `defaultSchema` を基準に `id` と `className` 属性を追加許可
- `<script>` タグ・イベントハンドラ属性（`onerror` 等）・`javascript:` スキームの href はすべて除去

### 9.2 クライアントサイド XSS 防止

検索結果の動的 HTML 生成では、タイトル・説明文・タグをエスケープしてから `innerHTML` に挿入する。
